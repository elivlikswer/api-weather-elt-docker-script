# api-weather-elt-docker-script
Данное приложение представляет из себя ETL Pipeline для сбора метеорологических данных из публичного API (https://open-meteo.com/) с возможностью контейнеризации и выбора формата хранения (CSV или parquet).
## Архитектура
Схема работы следующая: данные извелкаются из внешнего источника по запросу, в данном случае из публичного API Open-Meteo (**extract**), после полученные данные трансформируются в объект, с которым можно работать в рамках выбранной системы, в данном случае объект после преобразования становится объектов класса pandas (библиотека Python)  (**transform**), далее преобразованный объект с данными загружается локально на хост (**load**). Параметры, по которым строится объект данных - определяется при помощи конфигурационного файла .env. Пример конфигурационного файла приложен в репозитоии (.env.example). Также в репозитории предоставлена схема работы прилоежния (. 
## Технологии
Язык программирования: Python 3;
API: Open-Meteo API (https://open-meteo.com/);
Библиотеки: pandas, tenacity, poetry;
Docker для контейнерезации приложения
## Установка
> [!IMPORTANT]
> Для установки зависимостей необходимо установить Poetry.
### Локальный запуск.
1. Клонировать репозиторий
```bash
git clone https://github.com/elivlikswer/api-weather-elt-docker-script.git
```
2. Установка зависимостей 
```bash
poetry install
```
3. Настройка переменных окружения
Создайте .env из шаблона:
```bash
cp .env.example .env
```
4. Запуск
```bash
poetry run python src/main.py
```
### Запуск через Docker
1. Собрать образ
```bash
docker build -t weather-app .
```
2. Запустить контейнер
```bash
docker run --env-file .env \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logfiles:/app/logfiles \
  weather-app
```
## Переменные окружения
### LATITUDE
Широта местоположения. Положительные значения - северная широта, отрицательные - южная. Формат: число с плавающей точкой.
### LONGITUDE
Долгота местоположения. Положительные значения - восточная долгота, отрицательные - западная.Формат: число с плавающей точкой.
### BASE_URL
Адрес API для получения погодных данных. Используется сервис Open-Meteo. (пример: https://api.open-meteo.com/v1/forecast)
### START_DATE
Начальная дата периода, за который загружаются данные. Дата включительно. Формат: YYYY-MM-DD
### END_DATE
Конечная дата периода, за который загружаются данные. Дата включительно. Формат: YYYY-MM-DD
### OUTPUT_FORMAT
Значение по умолчанию: parquet
Доступные значения: csv, parquet
Описание: Формат файла для сохранения данных. csv - текстовый формат, удобный для просмотра в Excel. parquet - колоночный формат, эффективный для хранения и обработки больших объёмов данных.
### HOURLY
Значение по умолчанию: temperature_2m,relative_humidity_2m,rain,wind_speed_10m
Формат: список строк через запятую
Описание: Набор почасовых метеорологических параметров, которые будут загружены из API. Список доступных параметров можно найти в [документации Open-Meteo](https://open-meteo.com/en/docs).
### TIMEZONE
Значение по умолчанию: UTC
Часовой пояс, в котором будут возвращены временные метки.
